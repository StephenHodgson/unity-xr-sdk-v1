Detailed Description

Event handler implemented by a plugin for display subsystem graphics-thread specific lifecycle events.

All event callbacks will be executed on the graphics thread.

void* UnityXRDisplayGraphicsThreadProvider::userData

Plugin-specific data pointer. Every function in this structure will be passed this data by the Unity runtime. The plugin should allocate an appropriate data container for any information that is needed when a callback function is called.

◆ PopulateNextFrameDesc

UnitySubsystemErrorCode( * UnityXRDisplayGraphicsThreadProvider::PopulateNextFrameDesc) (UnitySubsystemHandle handle, void *userData, const UnityXRFrameSetupHints *frameHints, UnityXRNextFrameDesc *nextFrame)

PopulateNextFrameDesc will be called after simulation and pending graphics calls have completed PopulateNextFrameDesc is responsible for:

  • waiting for cadence (waiting until unity should start submitting rendering commands again) (if SubmitCurrentFrame didn't block)
  • acquiring the next image and telling Unity what texture id to render to next via nextFrame.
Parameters
[in]handleHandle obtained from UnityLifecycleProvider callbacks.
[in]userDataUser specified userData that was filled out on UnityXRDisplayGraphicsThreadProvider
[in]frameHintsInformation about the frame that was just rendered.
[out]nextFrameInformation that must be filled out about the next frame to be rendered.
