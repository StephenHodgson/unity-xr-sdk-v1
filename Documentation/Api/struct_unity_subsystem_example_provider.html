Detailed Description

Example container illustrating the interface of functions for Unity to call into a plugin. A plugin should initialize all of the (relevant) function pointers with function addresses local to the plugin library, and then return the structure to Unity in the plugin's Initialize function using the RegisterProviderFunctions function that's available as part of the Unity plugin interface structure (below).

void* UnitySubsystemExampleProvider::userData

Plugin-specific data pointer. Every function in this structure will be passed this data by the Unity runtime. The plugin should allocate an appropriate data container for any information that is needed when a callback function is called.
