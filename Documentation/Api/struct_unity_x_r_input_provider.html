Simple, generic method callback to inform the plugin or individual devices of events occurring within unity

Parameters
[in]handleHandle obtained from UnityLifecycleProvider callbacks.

[in]eventTypeThe type of event being sent
[in]deviceIdThe Id of the device that should handle the event. Will be kUnityInvalidXRInputFeatureIndex if event is for all devices.

[in]sizeThe size of the buffer payload. This can be used to verify that the type is actually what was expected.

◆ HandleHapticImpulse

UnitySubsystemErrorCode( * UnityXRInputProvider::HandleHapticImpulse) (UnitySubsystemHandle handle, void *userData, UnityXRInternalInputDeviceId deviceId, int channel, float amplitude, float duration)

A call from Unity when the developer requests a haptic impulse or rumble. This should only be processed if the device reports that it supports haptic impulses via QueryHapticCapabilities.

Parameters
[in]handleHandle obtained from UnityLifecycleProvider callbacks.
[in]userDataUser specified data.
[in]deviceIdThe input device that the message is being directed to.
[in]channelThe haptic channel that the impulse is requested for.
[in]amplitudeThe requested amplitude in a [0,1] range.
[in]durationThe >0 duration that the impulse is requested for.
Returns
kUnitySubsystemErrorCodeSuccess The provider will trigger the haptic impulse on the specified device.
kUnitySubsystemErrorCodeFailure The provider will not be triggering the haptic impulse.
